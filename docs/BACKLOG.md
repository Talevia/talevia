# Backlog

`/iterate-gap` 的驱动源。每个 cycle 的流程：

1. 读这个文件，挑**最靠上的未完成**任务（P0 → P1 → P2，同档内按出现顺序）。
2. plan → 实现 → 验证 → 归档决策 → commit + push。
3. 在 `docs(decisions)` 那次 commit 里顺手把本任务的 bullet 从这个文件里删掉（不是打钩，直接删），保持列表紧凑。
4. 如果这个文件的 P0/P1/P2 三档都已清空（或文件不存在），`/iterate-gap` 会按 skill 里的 rubric 分析一次性生成 **20 条新任务**写回这个文件、commit，然后**在同一个 cycle 里继续**挑新生成列表里第 1 条开干。

任务描述只给 **Gap（现状 / 痛点）** 和 **方向（期望动的东西）** —— 细节留给 cycle 里的 plan 阶段现读代码决定。Rubric 轴用 `§5.x` 标。

## P0 — 高杠杆、下一步就该动


## P1 — 中优，做完 P0 再排

- **source-query-roots-filter** — `source_query(select=nodes)` has no "only root nodes" / "only leaves" filter. `dag_summary` surfaces `rootNodeIds` as one field on a single summary row; iterating them needs a second dispatch. **方向：** add `hasParent: Boolean? = null` to the `nodes` select — `true` returns nodes with ≥1 parent (children-of-something), `false` returns roots only, `null` unchanged. Preserves the single-dispatcher shape; no new tool. Rubric §5.1.
- **agent-pre-compaction-tool-archive** — When Compactor prunes `Part.Tool(Completed)` outputs + writes a 200-word `Part.Compaction` summary, the original tool payloads are lost. Operators can't answer "what exactly was the output of `ExportTool` that just got compacted?". **方向：** before marking each tool part compacted, archive its `outputForLlm` + `data` payload to `<bundle>/session-compaction-archive/<sessionId>/<partId>.json`. `session_query(select=compaction_archive, partId=X)` retrieves. Parallels cycle-45's `<bundle>/source-history/` convention — bundle-local, git-tracked, recoverable across re-opens. Rubric §5.4.
- **permission-persistent-rules** — `DefaultPermissionRuleset` is built fresh at each AppContainer construction; interactive "remember for this session" replies evaporate on process restart. OpenCode persists via `permissions.storage` (see `packages/opencode/src/permission/index.ts`). CLI users re-answer the same prompts on every startup. **方向：** serialise per-user rule grants to `<userDataDir>/permission-rules.json`; `DefaultPermissionService` reads on init + writes on "remember" replies (with `revoke_permission` via a maintenance slash-command for operator housekeeping). Rubric §5.4.
- **debt-clip-render-cache-map-lookup** — `ClipRenderCache.findByFingerprint` is O(N) linear scan over `entries: List<ClipRenderCacheEntry>`. N stays under a few hundred on real projects today; per cycle-47's decision (`2026-04-23-debt-perclip-benchmark-inverted-scaling-under-stub-engine.md`), deep caches will eventually make lookup non-trivial — the benchmark already showed 1–2 ms/op lookup overhead swamping no-op render savings. **方向：** `entries` becomes `Map<String, ClipRenderCacheEntry>` keyed by fingerprint; "later entry wins" semantics preserved via `put`-overwrite. `findByFingerprint` becomes O(1). **触发条件：** profiling data showing `findByFingerprint` ≥ 5% of export wall time on a deep-cache project. Rubric 外.
- **bus-event-per-session-subscribe-helper** — Tests + tools filter bus events by sessionId repeatedly (~15 call sites). Each rewrites `.filter { it.sessionId == sid }` — AssetsMissing (project-scoped, not session-scoped) tests have occasionally forgotten this. **方向：** `EventBus.forSession(sessionId)` already exists; add a typed `sessionScopedSubscribe<E: SessionEvent>(sid)` that combines `filterIsInstance<E>()` + session filter with a compile-time `SessionEvent` constraint so non-session events fail to compile at the callsite. Current callers migrate one line. Rubric §5.6.
- **recents-registry-list-summaries-index** — `RecentsRegistry.listSummaries()` scans every entry + decodes every `talevia.json` envelope on every call. N ≥ several hundred projects means measurable per-call latency. **方向：** registry self-caches `(title, updatedAtEpochMs, createdAtEpochMs)`; envelope writes sync the cache; `listSummaries` reads the cache, skipping bundle I/O. **触发条件：** profiling data showing `listSummaries` ≥ 50 ms on real bundles. Rubric §5.3.
- **bundle-mobile-document-picker** — Android / iOS confined to app-sandbox bundles (`<filesDir>/projects/` / `Documents/projects/`). No way to pick an external bundle from SAF / Files.app. **方向：** Android wires `Intent.ACTION_OPEN_DOCUMENT_TREE`; iOS wires `UIDocumentPickerViewController`; both translate the returned URI / NSURL through a platform-specific resolver to an Okio `Path` that `FileProjectStore.openAt` accepts unchanged. **触发条件：** CLAUDE.md platform-priority window for mobile transitions from "don't regress" baseline to active development. Rubric §5.4 / mobile.
- **session-query-active-run-summary** — `session_query(select=status)` returns the current phase (Generating / AwaitingTool / etc.) but no running aggregate: "since this run started: N tool calls, M tokens in/out, P seconds elapsed, Q retries scheduled". Ops must cross-reference three selects. **方向：** new `session_query(select=active_run_summary, sessionId=X)` composes the running stats into one row from existing parts + bus-driven trackers. Read-side only; no state added. Rubric §5.4.
- **debt-register-tool-script** — Adding a new tool still needs appending to 4 AppContainers (CLI / Desktop / Server / Android; iOS skips registration). Shared `register*Tools(...)` extensions from cycle-37/46 help but new tools still need one-line additions in each extension function body. **方向：** `/register-tool <ToolName> <ctorArgs>` script that generates the cross-file diff + auto-runs `ktlintFormat`. **触发条件：** ≥ 10 consecutive new-tool cycles paying this tax (would show as many `register*Tools` diff-only patches in `git log` with no algorithmic substance). Rubric 外.

## P2 — 记债 / 观望

- **bundle-talevia-json-split** — `talevia.json` holds timeline + assets + source DAG + lockfile + snapshots in one file. Single mutation git-diff can grow to 100+ lines; snapshots could eventually make the file MB-scale. **方向：** split into `assets.json` / `timeline.json` / `lockfile.json` / `snapshots/<id>.json` sub-files; `talevia.json` envelope keeps only schemaVersion + metadata + sub-file manifest. **触发条件：** real user report of diff noise or snapshot file ≥ 1 MB. Rubric §3a-3.
- **debt-unified-dispatcher-select-plugin-shape** — Every new select on `project_query` / `session_query` / `source_query` / `provider_query` touches 7–8 coordinating sites (SELECT_* constant + ALL_SELECTS + Input field + helpText + JSON Schema + `rejectIncompatibleFilters` matrix + `execute` dispatch + sibling handler file). Reject-matrix cost is `O(n_selects × n_filters)`. **方向：** "select plugin" shape — one file per select, `@AppliesTo(SELECT_FOO)` annotation or similar drives filter applicability, collapses the 7–8 sites into 1. **触发条件：** any dispatcher hits ≥ 20 selects OR `rejectIncompatibleFilters` hits ≥ 30 rules. Current: project 14 / session 15 / source 6 / provider 3. Rubric 外.
- **debt-desktop-missing-asset-banner** — Cycle-3 landed CLI-side `BusEvent.AssetsMissing` warning rendering. Desktop's SnapshotPanel / ExportPanel still don't surface the same warning visually. **方向：** Desktop AppContainer subscribes to the bus event; SnapshotPanel / ExportPanel render a warning banner near the export action (count + up to 5 paths + overflow tail, matching CLI's notice). **触发条件：** Desktop reaches CLI parity (CLAUDE.md platform priority). Rubric 外.
- **debt-bundle-fs-testkit-copy-recursive** — Okio's `FakeFileSystem` lacks a recursive copy primitive. `BundleCrossMachineExportSmokeTest` inlines a 12-line `copyDirectoryRecursive(fs, src, dst)` helper. **方向：** when a second bundle-level test needs the same walker, extract to `ProjectStoreTestKit` or a new `BundleFsTestKit`; don't pre-abstract for N=1. **触发条件：** second caller appears. Rubric 外 / 顺手记录.
- **set-session-preferred-provider** — `ProviderRegistry.default` is deterministic (first provider wired at AppContainer construction). No per-session override — a session that wants Anthropic-default must rely on the AppContainer's order. **方向：** `set_session_preferred_provider(providerId)` tool stores preference on `Session`; `Agent.run` consults before falling through to `ProviderRegistry.default`. Fallback chain unchanged. **触发条件：** operator report that the AppContainer default picks the wrong provider for a specific session's needs. Rubric §5.2.
- **source-node-diff-restore-composite-tool** — Once the P0 `source-node-body-restore-from-history` lands, users will want a per-field merge-restore: "revert only the `character_ref.name` field of this node to revision 3, keep the other body fields". **方向：** `merge_source_node_body_from_history(nodeId, revisionIndex, fieldPaths: List<String>)` — resolves JSON pointer paths in the old revision + overlays only those onto the current body. **触发条件：** P0 restore tool lands + operator asks for per-field granularity. Rubric §5.5.
- **compaction-token-budget-per-model** — `Compactor.compactionTokenThreshold = 120_000` is one-size-fits-all across every model. Haiku-4.5 (200k ctx) and GPT-4o (128k) should compact at different thresholds — today a Haiku-backed session compacts ~40% too early. **方向：** thread `ModelRef` through `Compactor.process` / the threshold check; use `ProviderRegistry.modelInfo(model).contextWindow × 0.85` as the per-model default (fallback to existing 120k when the model isn't in the registry). Rubric §5.4.
- **cli-metrics-slash-command** — `CounterRegistry` exposes counters via server's `GET /metrics` only. CLI / Desktop operators can't inspect live counts to answer "is tool X firing more than expected?". **方向：** `/metrics` slash-command on CLI renders current counters as a 2-column table (name, value). No new metric types, just a read-out. **触发条件：** operator reports "can't tell if tool X is firing more than expected" on CLI. Rubric §5.4.
- **agent-provider-fallback-chain-tracking** — `BusEvent.AgentProviderFallback` is bus-only today. `session_query(select=run_failure)` (cycle-50) can't populate a `fallbackChain` field because there's no tracker capturing from/to provider hops post-process. **方向：** add `AgentProviderFallbackTracker` sibling to `AgentRunStateTracker` — subscribes to `AgentProviderFallback` via the bus, keeps a bounded ring buffer per session; `RunFailureRow` gains `fallbackChain: List<{fromProviderId, toProviderId, reason, epochMs}>`. Rubric 外 / 顺手记录.
- **agent-retry-attempt-tracker-capture** — `AgentRunStateTracker.StateTransition` predates cycle-41's `AgentRunStateChanged.retryAttempt` field; transition history discards the retry attempt count. `session_query(select=run_failure)` (cycle-50) can't populate `maxRetryAttemptObserved` as a result. **方向：** extend `StateTransition` with `retryAttempt: Int? = null` (backwards-compatible via default); update the tracker's collector to capture it; extend `RunFailureRow` with `maxRetryAttemptObserved`. Rubric 外 / 顺手记录.
