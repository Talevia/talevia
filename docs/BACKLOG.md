# Backlog

`/iterate-gap` 的驱动源。每个 cycle 的流程：

1. 读这个文件，挑**最靠上的未完成**任务（P0 → P1 → P2，同档内按出现顺序）。
2. plan → 实现 → 验证 → 归档决策 → commit + push。
3. 在 iteration 的 commit 里顺手把本任务的 bullet 从这个文件里删掉（不是打钩，直接删），保持列表紧凑。
4. 如果这个文件的 P0/P1/P2 三档都已清空（或文件不存在），`/iterate-gap` 会按 skill 里的 rubric 分析一次性生成 **20 条新任务**写回这个文件、commit，然后**在同一个 cycle 里继续**挑新生成列表里第 1 条开干。

任务描述只给 **Gap（现状 / 痛点）** 和 **方向（期望动的东西）** —— 细节留给 cycle 里的 plan 阶段现读代码决定。Rubric 轴用 `§5.x` 标。

## P0 — 高杠杆、下一步就该动

- **debt-aigc-tool-consolidation** — 当前 4 个 AIGC 工具 (`generate_image` / `generate_video` / `generate_audio` / `synthesize_speech`) 各自独立 dispatcher。每个 ~700–1000 token tool spec，audit-subset 的最大单点开销 (project_query=998t, clip_action=996t 之后第二梯队)。User authorization 2026-05-02 解除"无法删旧工具因 LockfileEntry.toolId stamped"卡点 — 不要兼容性。**方向：** 合并到单 dispatcher tool `aigc_generate(kind: enum {image | video | audio | speech}, prompt, params)`. `LockfileEntry` 加 `kind` 一等字段，`toolId` 改 stamp `"aigc_generate"`. 老 lockfile entry 在新代码下不可 replay (acceptable per 无兼容性). 预期 audit-subset 下降 ~700–1000 tokens. Rubric §5.6 / §5.7 / §3a #1。Milestone §later.
- **lockfile-incremental-load** — 当前 `FileProjectStore.openAt` 全量 JSON-decode `Project.lockfile.entries`（O(N) at open time，每次 mutate 后整块 re-encode），随项目寿命线性恶化。User authorization 2026-05-02 解除"multi-file write atomicity 待用户决定"卡点 — best-effort atomicity OK. **方向：** 拆 lockfile 出 `talevia.json`，作为独立 `<bundleRoot>/lockfile.jsonl`（每行一条 entry，append-only）. `talevia.json` 只剩 envelope + timeline + source + cache + snapshots. 写顺序: 先 fsync `lockfile.jsonl`，再 fsync `talevia.json` (best-effort atomicity); crash recovery 用 `lockfile.jsonl` 完整性扫描 — 末尾不完整一行 truncate 即可. 读路径 lazy: 启动时只读 envelope，`findByInputHash` / `findByAssetId` 等查询时按需流式扫 + 内存索引. Rubric §5.7 / §5.3 / §3a-3。Milestone §later.
- **aigc-result-multi-variant** — 当前每次 AIGC 调用单 variant 输出. provider 支持 `n` (OpenAI image gen) 时浪费, 不支持 (Sora 2 / Veo) 时无法 batch；用户拿到一份 generation 不能 "give me 4 takes, pick the best"。User authorization 2026-05-02 — variant-key 设计已定. **方向：** `LockfileEntry.variantIndex: Int = 0` 一等字段；`inputHash` 计算时**包含** variantIndex（避免 N variants 共用 hash 抢占缓存）；provider 支持 `n` 的 一次返回 N 个 variant + N 条 lockfile entry；不支持的 顺序 N 次调用 + 透明 billing；UI / `session_query(select=spend_summary)` 显示 N × cost. AIGC tool input 加 `variantCount: Int = 1` 参数；output 升级为 `List<{assetId, variantIndex, costCents}>`. Rubric §5.2。Milestone §later.
- **debt-split-session-action-tool-input** — `SessionActionTool.kt` 606 LOC, `Input` 是 17 个 optional 字段大杂烩（每个 verb 一组字段）. User authorization 2026-05-02 解除"big-bang 风险高"卡点 — 不要兼容性. **方向：** sealed `Input` + 每 verb 一个 data class（`SessionActionTool.ArchiveInput / RenameInput / DeleteInput / ForkInput / RevertInput / CompactInput / ListInput / ...` 共 14 个）；用 kotlinx.serialization `@JsonClassDiscriminator("action")` 让 LLM JSON Schema 用 oneOf；一次性 big-bang refactor (handler signatures + 13 test files 同 cycle 内更新)；破坏 `Part.Tool.input` 已持久化 JSON (旧 session 无法 decode — acceptable).
