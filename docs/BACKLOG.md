# Backlog

`/iterate-gap` 的驱动源。每个 cycle 的流程：

1. 读这个文件，挑**最靠上的未完成**任务（P0 → P1 → P2，同档内按出现顺序）。
2. plan → 实现 → 验证 → 归档决策 → commit + push。
3. 在 iteration 的 commit 里顺手把本任务的 bullet 从这个文件里删掉（不是打钩，直接删），保持列表紧凑。
4. 如果这个文件的 P0/P1/P2 三档都已清空（或文件不存在），`/iterate-gap` 会按 skill 里的 rubric 分析一次性生成 **20 条新任务**写回这个文件、commit，然后**在同一个 cycle 里继续**挑新生成列表里第 1 条开干。

任务描述只给 **Gap（现状 / 痛点）** 和 **方向（期望动的东西）** —— 细节留给 cycle 里的 plan 阶段现读代码决定。Rubric 轴用 `§5.x` 标。

## P1 — 中优，做完 P0 再排

- **aigc-result-multi-variant** — `generate_image` / `generate_video` 每次产 1 个 asset；OpenAI / Replicate 都支持 `n` 参数返 N 个候选让人挑。当前 agent 拿不到。**方向：** AIGC tools 接收 `n: Int = 1`，返回 List<assetId>；lockfile 每个变体一条 entry，共享 inputHash + 不同 variantIndex。Permission ASK 一次性覆盖 N 次成本。Rubric §5.2。Milestone §later. · skipped 2026-04-25: §3a #4 + design ambiguity — bullet's "shared inputHash" semantic conflicts with seed-in-hash invariant (variants need different seeds → different hashes), AND video providers (Sora 2 / Veo) don't natively support n so per-variant calls have separate-billing UX implications the user should approve before shipping.
- **effect-kind-registry-cross-engine-tax** — §5.2 compiler 轴信号：每加一个 filter / transition / overlay kind，要在 FFmpeg + Media3 + AVFoundation 三套 engine 各写一遍 (CLAUDE.md "Known incomplete" 已确认 vignette / transitions / subtitles / LUT 这套对齐税)；下一个新 kind 的 cross-engine 落地成本是 `≥ 3 platforms × ≥ 1 file each`。**方向：** core 层提供 `VideoEffectKind` 注册表 + `EffectHandler<TConfig>` 接口，每个 engine 实现一次 dispatch loop 而不是 per-kind switch；core 一处加 kind 后，三 engine 实现"找不到 kind" → fail loud 的同质 fallback。把跨平台对齐从手工责任翻成 compiler 检查到的接口契约。Rubric §5.2 / §5.6。Milestone §later. · skipped 2026-04-25: scope — 3-platform refactor without a concrete new effect kind to validate the registry against; current 4 aligned kinds work via existing test coverage. Better dispatched alongside the next new effect-kind addition where the registry shape will be exercised by a real new caller.
- **desktop-merge-conflict-strategy** — §5.4 multi-user 信号：bundle 已 git-friendly，`talevia.json` 是 pretty-printed JSON，但两人同时改相邻 clip / 同一段 source node body，git merge 会产生文本冲突，用户要手撸 JSON。**方向：** 提供 `talevia merge --base <ref> --ours <ref> --theirs <ref>` 命令，理解 timeline / source / lockfile 三块的合并语义（timeline = clip-id-keyed merge；source = node-id 替换冲突探测；lockfile = 永远 union）。或最少做出 `git merge` driver 在 `.gitattributes` 里注册成 `merge=talevia-json`。Rubric §5.4 / §3.4。Milestone §later. · skipped 2026-04-25: scope — new top-level `talevia merge` CLI command with semantic 3-way merge across timeline (clip-id-keyed) + source (node-id) + lockfile (union) is multi-cycle work; no real-world multi-user bundle conflict reported yet to baseline against.
